# Selected level record flag follow-up

Interpreted names:
- `+0xff25d0` -> `selected_level_record_active`
- `+0xff25d1` -> `selected_level_record_persistent`
- `+0xff25d4` -> `selected_level_record`

## Writers

- `0x409a1e` / `0x409a4b` | `update_galaxy`
  - `selected_level_record_active = 1`
  - `selected_level_record = base + route_index * 0x1fac0 + 0x944150`

- `0x4166a3` / `0x4166b7` | `update_challenge_setup_screen`
  - `selected_level_record_active = 1`
  - `selected_level_record = fixed challenge entry`

- `0x4165c7` / `0x4165ed` | `update_challenge_setup_screen`
  - `selected_level_record_active = 1`
  - `selected_level_record = base + level_mode_arg * 0x1fac0 + 0x944150`

## Consumers

- `0x435df8` .. `0x435e03` | `set_subgame_features`
  - if `selected_level_record_active != 0`, feature colour/metadata comes from `selected_level_record + 0x38`

- `0x435ec7` / `0x435ecf` | `populate_runtime_track_cells_from_segments`
  - when `selected_level_record_active != 0`, the seed/high-score path uses `selected_level_record + 0x68`

- `0x437fb7` .. `0x438148` | `build_subgame_level`
  - if `selected_level_record_active != 0 || selected_level_record_persistent != 0`, copy mode/rate/course fields from `selected_level_record`

- `0x438b02` .. `0x438b13` | `update_subgame`
  - clears `selected_level_record_persistent` on frontend entry when it was set

- `0x438c6d` .. `0x438c8f` | `update_subgame`
  - if `selected_level_record_persistent == 1`, temporarily re-arm `selected_level_record_active = 1`, rebuild from `selected_level_record + 0x28`, then clear `selected_level_record_active` on the normal frontend leg

- `0x43ca01` | `update_subgoldy`
  - gameplay still reads `selected_level_record_persistent` during the death/completion path

## Interpretation

- There is enough evidence now to reject tutorial-only or replay-only naming.
- The cluster is a frontend-selected level-record override path that can persist across rebuild/front-end transitions.
- Exact-offset sweeps in the refreshed full HLIL export still only recover the known `build_subgame_level`, `update_subgame`, and `update_subgoldy_resurrect` references.
- The exact writer for `selected_level_record_persistent` is still unresolved in this slice.
